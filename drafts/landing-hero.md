# Instrument, not the answer

Most of what gets called AI-native is just putting a model where code used to be: a function becomes a prompt, a rule becomes a judgment call. That is not imagination. It is substitution, and it is usually the worst trade on the table.

Deterministic, idempotent code is low-entropy by construction. AI is high-entropy by design. That is its gift for writing the mechanical parts and its disqualification for being one. The same model is a superpower or a liability depending on whether its output is the instrument or the answer. Use it to build the rails. Do not make it run on them.

racecar is that conviction, executable: deterministic, idempotent checks an agent loads and runs against the code it just wrote. No model in the verification path.
